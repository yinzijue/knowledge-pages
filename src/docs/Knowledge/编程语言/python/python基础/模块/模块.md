# 概述
前面提到函数，是一段可复用的代码，是对功能的封装。而python中的模块则是更大范围的代码封装。
简单来讲，python中的模块是一个个.py文件。模块中可以是各种python对象的定义，也可以是若干个功能各异的函数。使用时，你需要现将这些模块导入，之后你就可以访问其中的变量、对象，或是使用其中的函数。

# 创建模块

编写模块有各种各样的方法，作为初学者，最简单的方式就是创建以.py为扩展名的文件，然后在该文件中编写相关变量、函数等。

```python
# Fibonacci number module
def fib(n): #write Fibonacci series up to n
    a,b = 0,1
    while a<n:
        print(a,end='')
        a,b = b, a+b
    print()
    
def fib2(n): # return Fibonacci series up to n
    result = []
    a,b = 0,1
    while a<n:
        result.append(a)
        a, b = b,a+b
    return result
```

# 使用模块

## 导入模块

若你想在你的某个程序中使用某个模块，那么你至少要告诉你的程序这个模块的所在路径以及它的名字。如果你需要频繁的使用到其中的某个函数，为了之后编写方便，你甚至可以在导入时告诉你的程序模块中存在哪些函数。

python中的模块有三种来源：标准模块(python自带)、第三方模块(导入前需要先安装)、自定义模块。

导入模块的格式大致分为两种: **import...** 和**from...import...**。

### import packagename.modulename
- **这种方式只导入了包名和模块名**，若要使用其中的函数或变量，需要使用绝对引用，即packagename.modulename.functionname

- 标准模块可以直接导入；
- 当需要导入的模块与当前可执行的程序同属一个包时，包名可以省略；
- 可以使用 import modulename1,modulename2...的方式一次性导入多个模块；

### from packagename.modulename import functionname,variable

- 这种方式可以导入模块中的指定函数、全局变量等，而无需导入全部模块；
- 导入的函数和变量可以直接使用，而无需带上包名和模块名；
- 导入模块、函数等内容时，可以使用as关键字为其创建别名，方便后续引用，这在导入内容的名称很长时很有用。
```python
from moduleName import functionName as alia
```
- 对于from...import...的导入形式，可以使用**from modulename import ***的方式导入**模块中所有不以下划线开头的内容**。但使用这种方式前请确保你对导入模块的内容十分了解，因为导入未知的名称可能会覆盖当前程序已定义的名称。



## 包(package)

在学习模块导入的时候提到，部分模块在导入时需要带上包名。那么包在python项目中处于什么位置呢，在了解这点之前，我们先看看一个常规的python项目文件的层级结构。
![[python项目结构.png]]

最外层，是python的项目文件夹，项目之下，可以是目录、包，也可以是python文件。而目录和包之下又可以嵌套目录和包，或是python文件。

那么包和目录有什么区别呢？简单来说，包和目录的一个显著区别在于一个__init__.py文件，这个文件具有以下四个性质：

- 其本质是一个模块；
- 这个模块的模块名不是__init__，而是其所在包的名字；
- 它的作用是将一个文件夹变成一个python模块；
- 这个文件可以不包含任何代码内容，也可以写一些初始化代码，这些代码将在包被导入时自动执行；

>[!tip]+
>你可以尝试在pycharm中创建一个包，然后删除其中自动创建的__init__.py文件，观察包的图标变化，感受下包和目录的直观区别；

>[!important]+
>重要为了**快速加载**模块，我们常常会对模块文件进行编译，编译后生成.pyc文件。为了减小编译后模块文件的大小，可以在python命令中添加-O(去除断言语句)和-OO(去除断言语句和__doc__字符串)开关。但有些程序可能依赖于这些内容，因此在使用这两个选项时，请确保你对目标程序足够了解。
