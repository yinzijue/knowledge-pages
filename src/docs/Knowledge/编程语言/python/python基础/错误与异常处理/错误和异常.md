# 概述

正如平常写作一样，写程序也会犯错。python中定义了两种错误类型：句法错误(**syntax errors**)和异常(**exceptions**)。

如果你在前面的学习中曾尝试独立编写一些程序，相信你已经见过他们其中的一些了。

# 句法错误

句法错误，又称解析错误(parsing errors)。句法错误大概是你在学习python过程中最常见的一种错误类型了。但这类错误也比较容易被发现，目前市面上一些相对智能的IDE往往会在编码期间就帮你找出这些错误，而不是等到程序运行的时候。
![[python句法错误.png]]

如果你将鼠标悬停在这些错误之上，可以得到一些有用的建议，而如果你仍然试图运行这段代码，就可以在控制台中看到更详尽的错误信息。提示信息包括：

- 错误所在文件路径；
- 错误在文件中的位置(行数line)；
- 错误在语句中首次出现的地方(一个红色的向上的小箭头标识(token))
- 错误类型：句法错误(syntax error)
![[pycharm控制台错误信息.png]]

# 异常

还有一些错误，在编码期间，通常不会被IDE检测出来，而你本身也可能没注意到它们。只有当你试图运行包含这个错误的程序时，它们才会被触发。在程序执行期间检测到的错误被称为**异常**(exceptions)。异常不一定会引起严重错误，大多数也不会被程序自行处理，而是显示错误信息(抛出异常)。

## 异常处理

```python
try:
    statement1							 # 可能引发异常的语句
    statement2							 # 可能引发异常的语句
except XXException/xxError as alias:     # 捕获异常
    exception_handle statements
else:
    statements
finally:								 # 清理操作
    clean_up statements
```

### try

try语句的工作原理如下：

- 首先，执行try子句(try和except关键字之间的语句)；
- 如果没有触发异常，则跳过except子句，try语句执行完毕；
- 如果在执行try子句时发生了异常在，则跳过该子句中剩下的部分。如果异常的类型与except关键字后指定的异常相匹配，则会执行except子句，然后跳到try/except代码块之后继续执行；
- 如果发生的异常与except子句中指定的异常不匹配，则它会被传递到外部的try语句中；如果没有相应的处理程序，则它是一个未处理异常且执行将终止并输出内置的异常提示信息；
![[python异常处理流程]]

### except

**except语句是可选的，** try/finally风格下就没有except语句。

- 一个try语句可以有多个except子句来为不同的异常指定处理程序。但**最多只有一个处理程序会被执行**；
- 处理程序只处理对应的try子句中发生的异常，而不处理同一try语句内其他处理程序中的异常；
- except子句可以用**带圆括号的元组**来指定多个异常。

```python
except(RuntimeError,TypteError,NameError):
	pass
```
>[!info]+
>A class in an [except](https://docs.python.org/3/reference/compound_stmts.html#except) clause is compatible with an exception if it is the same class or a base class thereof (but not the other way around — an _except clause_ listing a derived class is not compatible with a base class) ---------[官网原话链接](https://docs.python.org/3/tutorial/errors.html#:~:text=A%20class%20in%20an%20except%20clause%20is%20compatible%20with%20an%20exception%20if%20it%20is%20the%20same%20class%20or%20a%20base%20class%20thereof%20\(but%20not%20the%20other%20way%20around%20%E2%80%94%20an%20except%20clause%20listing%20a%20derived%20class%20is%20not%20compatible%20with%20a%20base%20class\))

如果except子句中列出的异常类是和捕获的异常所属的类是相同的类，或者是其(*这个其，指捕获异常所属的类*)基类，则该类与该异常兼容(但反过来则不行——一个列出派生类的except子句和基类不兼容)。

```python
class B(Exception):			
    pass

class C(B):                 # C派生自B
    pass

class D(C):					# D派生自C
    pass

for cls in [B, C, D]:
    try:
        raise cls()      # 依次触发类型为B、C、D的异常
    except D:			 # except中列出的类为D，try触发的D与其兼容
        print("D")
    except C:			 # except中列出的类为C，try依次触发的C、D与其兼容
        print("C")
    except B:			 # except中列出的类为B，try依次触发的B、C、D与其兼容
        print("B")
######################################################
"""
输出的结果为：
D
C
B
"""
```

```python
class B(Exception):			
    pass

class C(B):                 # C派生自B
    pass

class D(C):					# D派生自C
    pass

for cls in [B, C, D]:
    try:
        raise cls()      # 依次触发类型为B、C、D的异常
	# except中列出的类为B，try依次触发的B、C、D与其兼容，所有异常均可被其捕获
    except B:			 
        print("B")
    except C:			 
        print("C")
    except D:			 
        print("D")
##################################################
"""
输出的结果为：
B
B
B
"""
```

首先明确一点：在一个try子句触发的异常被捕获并处理后，try子句就结束了。

在错误顺序示例中，try 子句依次触发类型为B、C、D的异常，均和第一个except子句列出的类B兼容，因此该子句可以捕获所有的异常并处理。在自上至下的顺序执行下，下方的except子句没有机会执行。

#### 带参数的异常

except子句可以在异常名称后面指定一个变量，这个变量会绑定异常类的实例，而这个异常类的实例会有一个args属性用来存储这些参数。为了方便起见，内建的异常类型定义了__str__()方法用来打印所有的参数而无需显式地访问args属性。

异常参数常用于输出错误字符串，错误数字，错误位置等信息。

```python
try:
    raise Exception('spam', 'eggs')    # 构建一个异常实例，为其指定参数，并触发异常
except Exception as inst:
    print(type(inst))    # the exception instance
    print(inst.args)     # 通过访问args属性，查看异常参数
    print(inst)          
                         
    x, y = inst.args     
    print('x =', x)
    print('y =', y)
```

#### BaseException和Exception

- BaseException是所有异常类共同的基类；
- Exception(BaseException的子类之一)是所有非致命异常(non-fatal exceptions)类的基类；
- **非Exception子类**的异常通常不会被处理，因为这些异常通常被用于指示当前程序应当被终止；

>[!important]+
>Exception类可以作为一个通配符来捕获所有的异常。尽管这看上去很省力(无需判断异常所属类型)，但实际经验告诉我们，更好地处理方式应当是尽可能地明确我们即将要处理的异常的类型，避免真正的错误被掩盖。

### else

- try/except语句中，else子句是可选的；
- 一旦有else子句，它**必须出现在所有except子句之后**；
- 如果try子句执行时没有引发异常，python将执行else子句(如果else子句存在的话)；
- else子句常用于处理**try子句没有引发异常的情况下必须执行的操作**。
- <font color="red"> 相较于把代码追加到try子句中，更好的处理方式是使用else子句，这样可以避免意外捕获到未经try/except代码处理过的异常。</font>

### finally

try语句还有另一个可选的子句——finally子句，常用于定义在任何情况下都需要执行的清理操作,例如关闭网络连接、数据库连接等。如果存在finally子句，则finally子句将作为try语句完成之前的最后一个任务执行。无论try语句是否产生异常，finally子句都会执行。但实际应用中，还有一些更复杂的场景：

- 如果在执行try子句期间发生异常，则可以使用except子句处理该异常。如果异常没有由except子句处理，则在finally子句执行后重新引发异常；
- 在except子句和else子句执行期间也有可能产生异常。同样的，这些异常会在finally子句执行完成后被重新触发；
- **如果finally子句中执行了break、continue或者return语句，异常将不会被重新触发；**
- **如果try语句中执行了break、continue或者return语句，则finally子句的执行优先级将高于break、continue或return语句；**
- **如果finally子句中包含了return语句，则最终的返回值为finally子句中的返回值，而不是try子句中的返回值。**

```python
# try子句引发的NameError未被except子句处理，会在finally之后重新触发。
try:
    raise NameError
except ValueError:
    raise ValueError
finally:
    print('clean-up')
```

```python
# except子句和else子句在执行过程中分别引发了ValueError和IndexError，它们将在finally执行之后
# 被重新触发。
try:	
    raise NameError
except NameError as exc:
    raise ValueError from exc
else:
    raise IndexError
finally:
    print('clean-up')
```

```python
# finally子句中执行了break语句，try、except、else子句中的异常均不会被重新触发。
while True:
    try:
        raise NameError
    except NameError as exc:
        raise ValueError from exc
    else:
        raise IndexError
    finally:
        print('clean-up')
        break
```

```python
# finally子句中的break、continue、return优先级高于try子句中的。
def foo():
    try:
        a = 3
        return a
    finally:
        a = 4
        print(a)


print(foo())   # 输出结果为4 3
```

```python
# 最终返回finally子句中的返回值。
def bool_return():
    try:
        return True
    finally:
        return False

print(bool_return())         # 输出结果为False
```

### raise语句

当程序出错的时候，系统会自动引发异常。但有时会有一些无法被系统自动引发的异常，例如由于与业务需求不符产生的异常。此时就需要开发者来决定是否手动引发异常。

python提供了raise语句来实现强制引发指定的异常。

```python
raise [exceptionName [(reason)]]
```

- reason：关于异常的相关信息描述，reason是可选的，若省略，则抛出异常时不附带任何异常描述信息；
- exceptionName：指定要引发的异常的名称，exceptionName是可选的，若exceptionName和reason一起省略，则会将当前捕获的错误原样抛出；

raise语句有三种使用方式：

- 独立使用raise：该语句会引发当前上下文中捕获的异常(譬如except子句中)，或默认引发RuntimeError异常；
- raise 异常类：引发指定异常类型的异常；
- raise 异常类实例：异常实例可以携带属性，用以描述异常信息。

>[!tip]+
>手动引发异常通常时为了防止其崩溃。

```python
try:
    a = input("输入一个数：")
    #判断用户输入的是否为数字
    if(not a.isdigit()):
        raise ValueError("a 必须是数字")
except ValueError as e:
    print("引发异常：",repr(e))
```

### 异常链

如果在except子句处理异常的部分又引发了异常，python会将额外触发的异常附加到正在处理的异常中，并包含在错误信息中。

```python
try:
    open("database.sqlite")
except OSError:
    raise RuntimeError("unable to handle error")
######################################################################
#FileNotFoundError: [Errno 2] No such file or directory: 'database.sqlite'
#During handling of the above exception, another exception occurred:
#RuntimeError: unable to handle error
```

为了表明某个异常是另一个异常的直接结果，raise语句提供了一个额外的from子句，该子句会异常提示信息中直接指示异常之间的关系。

```python
def func():
    raise ConnectionError

try:
    func()
except ConnectionError as exc:
    raise RuntimeError('Failed to open database') from exc
##########################################################################
#ConnectionError
#The above exception was the direct cause of the following exception:
#raise RuntimeError('Failed to open database') from exc
#RuntimeError: Failed to open database
```

### 异常组(python 3.11)

在前面的学习中，我们得知，except子句列出多个异常时，最终只会有一个异常被处理。但在某些情况下，有必要报告已经发生的几个异常。在并发框架中经常出现这种情况，当多个任务可能同时失败时，其他的用例可能需要继续执行并收集多个错误，而不是引发第一个异常。

python 3.11提供了内建的ExceptionGroup类包装将多个异常实例包装成一个列表，以便将它们一并触发。此外，ExceptionGroup继承自Exception，其本身就是一个异常，因此异常组可以像其他异常一样被触发。

```python
from builtins import ExceptionGroup
def f():
    excs = [OSError('error 1'), SystemError('error 2')]
    raise ExceptionGroup('there were problems', excs)
f()

try:
    f()
except Exception as e:
    print(f'caught{type(e)}:e')                 # 异常类型为ExceptionGroup
```

通过使用except \* 来替代except，我们可以有选择地只处理组中匹配某个类型的异常。(个人推测可能是由于pycharm版本问题，在pycharm中except * 会提示此处不能使用星号表达式，实际上是可以正常运行的)

```python
from builtins import ExceptionGroup


def f():
    raise ExceptionGroup("group1",
                         [OSError(1),
                          SystemError(2),
                          ExceptionGroup("group2",
                                         [OSError(3), RecursionError(4)])])

try:
    f()
except* OSError as e:
    print("There were OSErrors")
except* SystemError as e:
    print("There were SystemErrors")
```
>[!attention]+
>需要注意的是，异常组中嵌套的异常必须使用异常实例而不是异常类。
