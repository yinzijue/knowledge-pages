# 定义类

最简单的类定义形式如下：

```python
class ClassName:
    <statement-1>
    .
    .
	.
    <statement-n>
```

# 类

python中万物皆对象。类本身也是对象。类支持两种操作：属性引用和实例化。

```python
class MyClass:
    """this is a doc string"""
    i = 12345
print(MyClass.i) 
print(MyClass.__doc__)
```

```python
# 创建一个空的实例对象，将其赋给局部变量var.
var = ClassName()
##########################################################
class MyClass:
    def __init__(self, name):
        self.name = name

# 创建一个带初始属性的实例对象
x = MyClass('jack')
```

- 属性引用使用标准的属性引用语法: obj.name，在上述示例中，则可以具体描述为ClassName.name
- 类实例化使用的函数表示法，可以把类对象看作一个无参函数，这个函数会返回一个该类的新实例。
- 可以在类中定义一个__init__()方法，用来创建一个带初始状态的实例对象，__init__()方法会在类实例化时自动调用；

# 实例

实例可以执行的唯一操作就是属性引用。实例有两种可用的属性名称：1）数据属性；2）方法；

## 数据属性

python中实例的数据属性，相当于Smalltalk中的实例变量，以及C++中的数据成员。数据属性不需要声明，在第一次被赋值时产生。

```python
class Complex:
    def __init__(self, realpart, imagpart):
        self.realpart = realpart
        self.imagpart = imagpart

x = Complex(3.0, -4.5)
```

## 实例方法

**方法是一个从属于对象的函数。** 在python中，方法并非实例独有，其他对象也有，例如之前接触到的列表对象就有append,insert等方法。但在此章节，我们只讨论实例方法。

根据定义，所有定义在类中的函数对象都是该类的实例对象对应的方法。

```python
class MyClass:
    i = 12345
	# 定义实例方法
    def foo(self):
        print('instance_method')


m = MyClass()
# 调用实例方法
m.foo()
print(type(Myclass.foo))
print(type(m.foo))
```

但MyClass.foo和m.foo并不是一回事，MyClass.foo是函数对象，而m.foo是方法对象。

### 方法对象

在运行上述示例时不难发现，当方法绑定实例后，该方法就会立即被调用。但立即调用一个方法并不是必须的。方法作为一种对象，可以被保存起来以后再调用。

```python
# 接上述示例
>>>x = m.foo
>>>x()
```

在此前的实践中，我们发现如果不带参数地调用一个需要参数的函数时，即使该参数实际没有被用到，也会引发异常。但在上述示例中，foo()方法指定了一个参数self,而在调用时，我们却并未显式的传递任何参数。

这是因为方法的特殊之处在于，实例对象会作为函数(实际是方法)的第一个参数被传入。因此，上述示例相当于：

```python
>>>MyClass.foo(m)
```

# 类变量和实例变量

通常来说，实例变量是独属于每个实例的数据，而类变量则被所有该类的实例所共享。

```python
class Demo:
    # 定义一个类变量
    cls_var = 'class variable'
	# 将传输方法的值赋给实例变量
    def __init__(self, ins_var):
        self.ins_var = ins_var

ins = Demo(ins_var='instance variable')
print(ins.ins_var)
print(ins.cls_var)
print(Demo.cls_var)
```

从上述示例中，我们可以看出：类变量可以被类和实例调用，但实例变量只能被实例调用。

>[!attention]+
>在使用类变量和实例变量时，有一点需要尤为注意：共享数据在涉及到可变对象时，可能会得到意料之外的结果。

```python
class Dog:
    tricks = []

    def __init__(self, name):
        self.name = name

    def add_tricks(self, skills):
        self.tricks.append(skills)


d = Dog('Fido')
e = Dog('Buddy')
d.add_tricks('roll over')
e.add_tricks('play dead')
print(d.tricks)                  # 输出结果为 ['roll over', 'play dead']
```

代码中分别添加给不同对象的属性，最后却在同一个对象中体现。

因此，在涉及可变对象时，更好的实践是将其定义为实例变量。

```python
lass Dog:

    def __init__(self, name):
        self.name = name
        self.tricks = []    # creates a new empty list for each dog

    def add_trick(self, trick):
        self.tricks.append(trick)
d = Dog('Fido')
e = Dog('Buddy')
d.add_trick('roll over')
e.add_trick('play dead')    
```
