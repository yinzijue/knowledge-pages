# 需求的可测试性

## 明确性
- 措辞精确
    - 无模糊性主观词汇：合理、适当等
- 逻辑闭环
    - 覆盖所有分支：正常流、异常流、边界

## 可观察性
- 结果是否可见
- 隐蔽逻辑如何验证：后台任务、算法推荐、加密处理

## 可控制性
- 前置条件是否容易构造
    - 会员3天到期弹窗提醒
- 异常场景是否可控
    - 网络超时、断网、宕机是否可以模拟

## 可隔离性
- 依赖是否可以解耦
    - 第三方依赖
        - 是否支持mock
    - 数据依赖
        - 依赖特定账号、需要特定历史数据积累...

## 数据可验证性
- 数据源是否可信
- 计算逻辑是否透明

## 环境依赖
- 硬件依赖
- 网络依赖
- 时间依赖
- ...
