# 你的第一个测试

Locust 测试本质上只是一个 Python 程序，它向你要测试的系统发送请求。这使得它非常灵活，特别擅长实现复杂的用户流程。但它也可以进行简单的测试，让我们从这里开始：

```python
from locust import HttpUser, task

class HelloWorldUser(HttpUser):
    @task
    def hello_world(self):
        self.client.get("/hello")
        self.client.get("/world")
```

这个用户会先向/hello发送一个HTTP请求，然后向/world发送请求，然后重复这个过程。要了解完整的解释和一个更现实的例子，请参阅“编写一个locustfile”。

将/hello和/world更改为你想要测试的网站/服务上的实际路径，将代码放入当前目录下的一个名为locustfile.py的文件中，然后运行locust：

```python
$ locust
[2021-07-24 09:58:46,215] .../INFO/locust.main: Starting web interface at http://0.0.0.0:8089
[2021-07-24 09:58:46,285] .../INFO/locust.main: Starting Locust 2.27.0
```

>[!note]+
性能测试结果的解读相当复杂（大部分超出了本手册的范围），但如果您的图表开始呈现这样的趋势，那么目标服务/系统无法承受当前的负载，您已经发现了一个瓶颈。
当我们达到大约9个用户时，响应时间开始迅速增加，尽管Locust仍在生成更多用户，但每秒的请求数却不再增加。这表明目标服务已经“过载”或“饱和”。
如果您的响应时间没有增加，那么您可以继续增加用户数量，直到找到服务的临界点，或者庆祝您的服务已经足够强大，能够处理您预期的负载。
如果您需要一些帮助来深入调查服务器端的问题，或者您正在努力生成足够的负载来使您的系统饱和，请查看Locust的常见问题解答（FAQ）。

# 直接命令行使用 /无界面模式


# 更多选项

要在多个Python进程或机器上分布式运行Locust，你需要首先使用 `--master` 命令行参数启动一个Locust主进程，然后使用 `--worker` 命令行参数启动任意数量的Locust工作进程。更多信息，请参见 Distributed load generation。

要查看所有可用选项，请输入：`locust --help` 或查看 Configuration。
