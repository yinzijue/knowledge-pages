# 自定义负载模式

有时需要一个完全自定义模式的负载测试，这种测试不能仅通过设置或改变用户数和生成率来实现。例如，你可能想要在某个自定义时间生成负载峰值或按照自定义的时间增加和减少负载。通过使用 LoadTestShape 类，你可以在任何时候完全控制用户数和生成率。

在你的 locustfile.py 文件中定义一个继承自 LoadTestShape 类的类。如果找到这种类型的类，Locust 将自动使用它。

在这个类中，你定义一个 tick() 方法，该方法返回一个包含所需用户数和生成率的元组（或者返回 None 来停止测试）。Locust 大约每秒调用一次 tick() 方法。

在类中，你还可以访问 get_run_time() 方法，以检查测试已经运行了多长时间。

# 示例

这个模式类将每 100 个用户增加用户数，然后在 10 分钟后停止负载测试：

```python
class MyCustomShape(LoadTestShape):
    time_limit = 600
    spawn_rate = 20

    def tick(self):
        run_time = self.get_run_time()

        if run_time < self.time_limit:
            # User count rounded to nearest hundred.
            user_count = round(run_time, -2)
            return (user_count, spawn_rate)

        return None
```

在 GitHub 上的示例中进一步展示了此功能，包括：

- 生成双波模式
- 类似 K6 的基于时间的阶段
- 类似 Visual Studio 的步进负载模式

对于你的自定义负载模式，可能还有一个有用的方法：get_current_user_count()，它返回当前活动的用户总数。这个方法可以用来防止在达到所需用户数之前进入后续步骤。这在每个用户的初始化过程很慢或耗时不确定时特别有用。如果你的用例听起来像这样，请查看 GitHub 上的示例。

请注意，如果你想要定义自己的自定义基础模式，你需要将其abstract属性定义为True以避免在导入时被识别为Shape：

```python
class MyBaseShape(LoadTestShape):
    abstract = True

    def tick(self):
        # Something reusable but needing inheritance
        return None
```

# 结合具有不同负载配置文件的用户

如果你使用 Web UI，你可以添加 --class-picker 参数来选择要使用的模式。但是，将你的用户定义放在一个文件中，而将 LoadTestShape 放在另一个文件中通常更加灵活。例如，如果你有一个在 low_load.py 中定义的低负载模式类，以及一个在 high_load.py 中定义的高负载模式类：

```bash
$ locust -f locustfile.py,low_load.py
$ locust -f locustfile.py,high_load.py
```

# 在每个时间间隔内限制要生成的用户类型

在返回值中添加 user_classes 元素可以为你提供更详细的控制：

```python
class StagesShapeWithCustomUsers(LoadTestShape):

    stages = [
        {"duration": 10, "users": 10, "spawn_rate": 10, "user_classes": [UserA]},
        {"duration": 30, "users": 50, "spawn_rate": 10, "user_classes": [UserA, UserB]},
        {"duration": 60, "users": 100, "spawn_rate": 10, "user_classes": [UserB]},
        {"duration": 120, "users": 100, "spawn_rate": 10, "user_classes": [UserA,UserB]},
    ]

    def tick(self):
        run_time = self.get_run_time()

        for stage in self.stages:
            if run_time < stage["duration"]:
                try:
                    tick_data = (stage["users"], stage["spawn_rate"], stage["user_classes"])
                except:
                    tick_data = (stage["users"], stage["spawn_rate"])
                return tick_data

        return None
```

# 在自定义模式中重用命令行参数

默认情况下，使用自定义模式将忽略默认的运行参数（在命令行界面和Web UI中）：--run-time、--spawn-rate、--users。

如果你需要其中一个或多个参数，你可以通过设置 use_common_options 属性为 True 来强制 Locust 接受它们：

```python
class MyCustomShape(LoadTestShape):

    use_common_options = True

    def tick(self):
        expected_run_time = self.runner.environment.parsed_options.run_time
        # Do something with this expected run time
        ...
        return None
```
