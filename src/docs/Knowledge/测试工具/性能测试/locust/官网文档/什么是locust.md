# 什么是Locust

Locust 是一个开源的性能/负载测试工具，支持HTTP和其他协议。其面向开发者的方法允许你使用常规的Python代码来定义测试。

Locust测试可以通过命令行或基于Web的用户界面来运行。吞吐量、响应时间和错误可以实时查看，也可以导出以供后续分析。

你可以将常规的Python库导入到你的测试中，并且由于Locust的可插拔架构，它具有无限的扩展性。与使用大多数其他工具不同，你的测试设计永远不会受到图形用户界面或特定领域语言的限制。

要开始使用Locust，请转到“安装”部分。

# 特点

- 在纯python环境下编写测试场景

如果你希望你的用户进行循环、执行一些条件行为或进行一些计算，你只需要使用Python提供的常规编程结构即可。Locust在每个用户内部都运行一个greenlet（轻量级进程/协程）。这使得你能够像编写正常的（阻塞式）Python代码一样编写你的测试，而无需使用回调函数或其他机制。因为你的场景就是“纯Python”代码，你可以使用你常用的IDE，并且像管理常规代码一样使用版本控制来管理你的测试（与其他使用XML或二进制格式的工具相反）。

- 分布式和可扩展——支持成千上万的并发用户

Locust 使得在多个机器上分布式运行负载测试变得简单。它是基于事件的（使用 gevent），这使得单个进程能够处理成千上万的并发用户。尽管在给定硬件上可能存在其他工具能够每秒执行更多的请求，但每个 Locust 用户的低开销使其非常适合测试高并发的工作负载。

- 基于Web的用户界面

Locust拥有一个用户友好的Web界面，可以实时显示你的测试进度。你甚至可以在测试运行时更改负载。它也可以在没有UI的情况下运行，使其易于用于CI/CD测试。

- 可测试任何系统

尽管Locust主要用于网站/服务，但它也可以用于测试几乎任何系统或协议。只需为你想要测试的内容编写一个客户端，或者探索社区创建的一些客户端。

- 可定制

Locust小巧且非常灵活，我们打算保持这一特点。如果你想要将报告数据发送到你喜欢的数据库和图形系统，封装REST API调用来处理你的系统的特定部分，或者运行一个完全自定义的负载模式，那么没有什么能阻止你！

# 名称与背景

Locust诞生于对现有解决方案的沮丧。没有现有的负载测试工具能够很好地为动态网站生成真实的负载，其中大多数页面为不同用户提供不同的内容。现有工具使用笨拙的界面或冗长的配置文件来声明测试。在Locust中，我们采取了不同的方法。你不会得到配置格式或UI，而是得到一个Python框架，它允许你使用Python代码定义用户的行为。

Locust的名字来源于蝗虫种类，它们以群居行为而闻名。
