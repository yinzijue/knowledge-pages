# 日志记录

Locust使用Python内置的日志记录框架来处理日志。

Locust应用的默认日志配置将日志消息直接写入标准错误输出（stderr）。可以使用--loglevel和--logfile来改变日志的详细程度或让日志输出到文件。

默认日志配置为根记录器（root logger）以及locust.*记录器安装了处理程序，因此如果你在自己的测试脚本中使用根记录器，并且使用了--logfile，那么消息将被写入日志文件：

```python
import logging  
logging.info("这条日志消息将会和其他Locust日志消息一起被记录")
```

你也可以通过在你的测试脚本中使用--skip-log-setup选项来控制整个日志配置。然后，你将需要自己配置日志。

# 选项

## --skip-log-setup

禁用Locust的日志设置。相反，配置由Locust测试或Python默认设置提供。

## --loglevel

选择DEBUG/INFO/WARNING/ERROR/CRITICAL中的一个。默认是INFO。短写形式是-L。

## --logfile

  
日志文件的路径。如果未设置，日志将输出到标准输出/标准错误。

# Locust记录器

以下是Locust内部使用的记录器列表（用于手动配置日志设置时参考）：

|   |   |
|---|---|
|**记录器名称**|**用途**|
|locust|locust命名空间用于所有记录器，如locust.main、locust.runners等。|
|locust.stats_logger|此记录器用于定期将当前统计信息打印到控制台。默认情况下，当使用--logfile时，统计信息不会写入日志文件。|
