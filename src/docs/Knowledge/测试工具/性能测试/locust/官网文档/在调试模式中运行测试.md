# 在调试模式中运行测试

在开发测试时，在调试器中运行 Locust 极其有用。除了其他功能外，您可以检查特定的响应或检查某些 User 实例变量。

但是，调试器有时在处理像 Locust 这样的复杂 gevent 应用程序时会出现问题，而且框架本身有很多您可能不感兴趣的内部活动。为了简化这一点，Locust 提供了一个名为 run_single_user 的方法：

```python
from locust import HttpUser, run_single_user, task


class QuickstartUser(HttpUser):
    host = "http://localhost"

    @task
    def hello_world(self):
        with self.client.get("/hello", catch_response=True) as resp:
            pass  # maybe set a breakpoint here to analyze the resp object?


# if launched directly, e.g. "python3 debugging.py", not "locust -f debugging.py"
if __name__ == "__main__":
    run_single_user(QuickstartUser)
```

它隐式地为请求事件注册了一个事件处理程序，以打印关于每个请求的一些统计信息：

```bash
type    name              resp_ms exception
GET     /hello            38      ConnectionRefusedError(61, 'Connection refused')
GET     /hello            4       ConnectionRefusedError(61, 'Connection refused')
```

您可以通过为 run_single_user 指定参数来精确配置要打印的内容。

请确保您已在调试器设置中启用了 gevent。在 VS Code 的 launch.json 中，它看起来像这样：

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Run current file",
            "type": "python",
            "request": "launch",
            "program": "${file}",
            "console": "integratedTerminal",
            "gevent": true
        },
        {
            "name": "Run locust",
            "type": "python",
            "request": "launch",
            "module": "locust",
            "args": [
                "-f",
                "${file}",
                "--headless",
                "--users=5"
            ],
            "console": "integratedTerminal",
            "gevent": true
        }
    ]
}
```

如果您想要整个 Locust 运行时环境（包括渐增负载、命令行解析等），您也可以这样做：

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Locust: 5 users, with specific config file",
            "type": "python",
            "request": "launch",
            "module": "locust",
            "args": [
                "-f",
                "${file}",
                "--headless",
                "--users=5",
                "--config=${fileDirname}/../locust.conf"
            ],
            "console": "integratedTerminal",
            "gevent": true
        }
    ]
}
```

在 PyCharm 中也有类似的设置。

>[!note]+
VS Code/pydev 可能会给您关于以下内容的警告：
sys.settrace() 不应在使用调试器时使用
这个警告可以安全地忽略（如果您知道如何消除这个警告，请告诉我们）。



您可以多次执行 run_single_user，正如在 debugging_advanced.py 中所示。

# 打印HTTP通信

有时，当一个 HTTP 请求在 Locust 中失败，但在常规浏览器/其他应用程序中工作正常时，理解失败原因可能会有些困难。以下是如何详细检查通信的方法：

对于 HttpUser（使用 python-requests）：

```python
# put this at the top of your locustfile (or just before the request you want to trace)
import logging
from http.client import HTTPConnection

HTTPConnection.debuglevel = 1
logging.basicConfig()
logging.getLogger().setLevel(logging.DEBUG)
requests_log = logging.getLogger("requests.packages.urllib3")
requests_log.setLevel(logging.DEBUG)
requests_log.propagate = True
```

对于 FastHttpUser（使用 geventhttpclient）：

```python
import sys
...

class MyUser(FastHttpUser):
    @task
    def t(self):
        self.client.get("http://example.com/", debug_stream=sys.stderr)
```
