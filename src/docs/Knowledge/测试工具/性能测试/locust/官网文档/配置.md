# 环境变量

选项也可以通过环境变量进行设置。它们通常与命令行参数相同，但首字母大写，并以 LOCUST_ 为前缀：

在 Linux/macOS 上：

```bash
$ LOCUST_LOCUSTFILE=custom_locustfile.py locust
```

在 Windows 上：

```bash
> set LOCUST_LOCUSTFILE=custom_locustfile.py
> locust
```

# 配置文件

选项也可以在 config 或 TOML 文件格式的配置文件中设置。

Locust 默认会查找 ~/.locust.conf、./locust.conf 和 ./pyproject.toml。你可以使用 --config 标志指定一个额外的文件。

```bash
$ locust --config custom_config.conf
```

以下是 Locust 支持的配置文件的快速示例：

locust.conf

```
locustfile = locust_files/my_locust_file.py  
headless = true  
master = true  
expect-workers = 5  
host = https://target-system  
users = 100  
spawn-rate = 10  
run-time = 10m  
tags = [Critical, Normal]
```

pyproject.toml

当使用 TOML 文件时，配置选项应在 [tool.locust] 部分内定义。

```toml
[tool.locust]  
locustfile = "locust_files/my_locust_file.py"  
headless = true  
master = true  
expect-workers = 5  
host = "https://target-system"  
users = 100  
spawn-rate = 10  
run-time = "10m"  
tags = ["Critical", "Normal"]
```

注意

配置值的读取（和覆盖）顺序如下：

~/.locust.conf -> ./locust.conf -> ./pyproject.toml -> （使用 --conf 指定的文件）-> 环境变量 -> 命令行参数

# 同时使用多个Locust文件

-f/--locustfile接受多个以逗号分隔的locustfiles。

示例：

具有以下文件结构：

```
├── locustfiles/
│   ├── locustfile1.py
│   ├── locustfile2.py
│   └── more_files/
│       ├── locustfile3.py
│       ├── _ignoreme.py
```

```bash
$ locust -f locustfiles/locustfile1.py,locustfiles/locustfile2.py,locustfiles/more_files/locustfile3.py
```

Locust将使用locustfile1.py、locustfile2.py和more_files/locustfile3.py

此外，-f/--locustfile选项也接受目录。Locust将在指定的目录中递归搜索*.py文件，并忽略以“_”开头的文件。

示例：

```bash
$ locust -f locustfiles
```

Locust将使用locustfile1.py、locustfile2.py和more_files/locustfile3.py

您还可以将-f/--locustfile用于网址。这将下载文件并将其用作普通的locustfile。

示例:

```bash
$ locust -f https://raw.githubusercontent.com/locustio/locust/master/examples/basic.py
```

# 使用带User类UI选择器的Locust运行

当使用--class-picker标志运行Locust时，您可以在WebUI中选择要运行的Shape类和User类。如果没有选择，则使用所有可用的User类。

示例：

具有以下文件结构：

```
├── src/
│   ├── some_file.py
├── locustfiles/
│   ├── locustfile1.py
│   ├── locustfile2.py
│   └── more_files/
│       ├── locustfile3.py
│       ├── _ignoreme.py
│   └── shape_classes/
│       ├── DoubleWaveShape.py
│       ├── StagesShape.py
```

```bash
$ locust -f locustfiles --class-picker
```

Web UI将显示：

![](https://cdn.nlark.com/yuque/0/2024/png/23188736/1715387001170-c84bfbc6-91b5-43ff-ae39-ce903a4faa2a.png)

类选择器还允许禁用单个用户任务、更改权重或固定计数，以及配置主机。

甚至可以为每个用户添加您希望可配置的自定义参数。只需在您的用户类中添加一个json类方法：

```python
class Example(HttpUser):  
    @task  
    def example_task(self):  
        self.client.get(f"/example/{self.some_custom_arg}")  
  
    @classmethod  
    def json(self):  
        return {  
            "host": self.host,  
            "some_custom_arg": "example"  
        }
```

# 用户类的配置

您可以在命令行上配置您可能希望为每个用户设置的任何设置，就像在UI中一样。使用--config-users参数，您还可以传递包含用户配置的JSON字符串或文件。要配置多个用户，您可以向--config-users传递多个参数或使用JSON数组。

每个用户设置对象必须包含一个key user_class_name。这个key对应于您希望配置的类。

示例：

```bash
$ locust --config-users '{"user_class_name": "Example", "fixed_count": 1}'
$ locust --config-users '[{"user_class_name": "Example", "fixed_count": 1}, {"user_class_name": "ExampleTwo", "fixed_count": 2}]'
$ locust --config-users '{"user_class_name": "Example", "fixed_count": 1}' '{"user_class_name": "ExampleTwo", "fixed_count": 2}'
$ locust --config-users config_users.json
```

# 统计设置的自定义

Locust统计的默认配置在stats.py文件的常量中设置。可以通过覆盖这些值来根据特定需求进行调整。为此，请导入locust.stats模块并覆盖所需的设置：

```python
import locust.stats  
locust.stats.CONSOLE_STATS_INTERVAL_SEC = 15
```

这可以直接在Locust文件中完成，或者提取到单独的文件中，以便所有Locust文件共同使用。

可以修改的统计参数列表包括：

|   |   |
|---|---|
|Parameter name|Purpose|
|STATS_NAME_WIDTH|控制台输出中请求名称列的宽度|
|STATS_TYPE_WIDTH|控制台输出中请求类型列的宽度|
|CSV_STATS_INTERVAL_SEC|如果配置了此选项，CSV文件写入的频率间隔（秒）|
|CONSOLE_STATS_INTERVAL_SEC|结果写入到控制台的频率间隔（秒）|
|CURRENT_RESPONSE_TIME_PERCENTILE_WINDOW|计算当前响应时间百分位时使用的窗口大小/分辨率（秒）|
|PERCENTILES_TO_REPORT|需要计算和报告的响应时间百分位列表|
|PERCENTILES_TO_CHART|UI图表界面中显示的响应时间百分位列表|
|MODERN_UI_PERCENTILES_TO_CHART|现代UI图表界面中显示的响应时间百分位列表|
|PERCENTILES_TO_STATISTICS|UI统计界面中显示的响应时间百分位列表|
