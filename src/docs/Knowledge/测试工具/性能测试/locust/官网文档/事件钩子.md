# 事件钩子

Locust 提供了许多事件钩子，可以用来以不同方式扩展 Locust。

例如，下面是如何设置一个事件监听器，它在请求完成后触发：

```python
from locust import events

@events.request.add_listener
def my_request_handler(request_type, name, response_time, response_length, response,
                       context, exception, start_time, url, **kwargs):
    if exception:
        print(f"Request to {name} failed with exception {exception}")
    else:
        print(f"Successfully made a request to: {name}")
        print(f"The response was {response.text}")
```

>[!note]+
在上面的例子中，使用了通配符关键字参数（**kwargs），它是空的，因为我们处理了所有参数，但这可以防止在未来 Locust 版本中添加新参数时代码出错。
同样，完全有可能实现一个客户端，它不为此事件提供所有参数。例如，非 HTTP 协议可能甚至没有 url 或响应对象的概念。你可以从监听器函数定义中移除任何缺失的字段，或使用默认参数。

当在分布式模式下运行 Locust 时，在运行测试之前，在工作节点上执行一些设置可能是有用的。你可以通过检查节点运行器的类型来确保你不在主节点上运行：

```python
from locust import events
from locust.runners import MasterRunner

@events.test_start.add_listener
def on_test_start(environment, **kwargs):
    if not isinstance(environment.runner, MasterRunner):
        print("Beginning test setup")
    else:
        print("Started test from Master node")

@events.test_stop.add_listener
def on_test_stop(environment, **kwargs):
    if not isinstance(environment.runner, MasterRunner):
        print("Cleaning up test data")
    else:
        print("Stopped test from Master node")
```

你还可以使用事件来添加自定义命令行参数。

要查看可用事件的完整列表，请参阅事件钩子（Event hooks）。

# 请求上下文

请求事件有一个上下文参数，它允许你传递关于请求的数据（如用户名、标签等）。可以在调用请求方法时直接设置它，或者在用户级别通过重写 User.context() 方法来设置。

从请求方法设置上下文：

```python
class MyUser(HttpUser):
    @task
    def t(self):
        self.client.post("/login", json={"username": "foo"})
        self.client.get("/other_request", context={"username": "foo"})

    @events.request.add_listener
    def on_request(context, **kwargs):
        if context:
            print(context["username"])
```

来自User实例的上下文：

```python
class MyUser(HttpUser):
    def context(self):
        return {"username": self.username}

    @task
    def t(self):
        self.username = "foo"
        self.client.post("/login", json={"username": self.username})

    @events.request.add_listener
    def on_request(context, **kwargs):
        print(context["username"])
```

从响应中的值设置上下文，使用 catch_response：

```python
with self.client.get("/", catch_response=True) as resp:
    resp.request_meta["context"]["requestId"] = resp.json()["requestId"]
```

# 添加Web路由

Locust使用Flask来提供Web UI，因此向Web UI添加web端点（endpoint）很容易。通过监听init事件，我们可以获取Flask应用实例的引用，并使用它来设置新的路由：

```python
from locust import events  
  
@events.init.add_listener  
def on_locust_init(environment, **kw):  
    @environment.web_ui.app.route("/added_page")  
    def my_added_page():  
        return "Another page"
```

你现在应该能够启动Locust并浏览到http://127.0.0.1:8089/added_page。请注意，它不会自动添加为一个新的标签页——你需要直接输入URL。

# 扩展Web UI

除了添加简单的web路由外，你还可以使用Flask的蓝图（Blueprints）和模板来扩展Web UI，不仅添加路由，还允许你展示与Locust内置统计信息并排的自定义数据。这更高级一些，因为它还涉及编写和包含由路由服务的HTML和Javascript文件，但可以极大地提高Web UI的实用性和可定制性。

一个扩展Web UI的完整工作示例，包括HTML和Javascript示例文件，可以在Locust源代码的示例目录中找到。

- extend_modern_web_ui.py：显示一个表格，其中包含每次调用的内容长度。
- web_ui_cache_stats.py：显示每次调用的Varnish命中/未命中统计信息。这可以很容易地扩展到其他CDN或缓存代理，并收集其他缓存统计信息，如缓存年龄、控制等。

# 为Web UI添加身份验证

Locust使用Flask-Login来处理身份验证。login_manager在environment.web_ui.app上公开，允许你灵活地实现任何你想要的身份验证方式！

要使用用户名/密码身份验证，只需为environment.web_ui.auth_args提供一个username_password_callback。你需要负责定义回调的路由并实现身份验证。

身份验证提供者还可以配置为允许从GitHub或SSO等第三方进行身份验证。只需提供所需auth_providers的列表。你可以指定按钮上显示的标签和图标。callback_url将是按钮指向的URL。你需要负责定义回调路由以及与第三方的身份验证。

无论你是使用用户名/密码身份验证、身份验证提供者还是同时使用两者，都需要为login_manager提供一个user_loader。user_loader应返回None以拒绝身份验证，或在应用程序应授予身份验证时返回一个User对象。

完整的示例可以在【身份验证】示例中看到。

# 运行后台greenlet

因为Locust文件“只是代码”，所以没有什么能阻止你生成自己的greenlet来与实际的负载/用户并行运行。

例如，你可以监控测试的失败率，并在其超过某个阈值时停止运行。

```python
import gevent
from locust import events
from locust.runners import STATE_STOPPING, STATE_STOPPED, STATE_CLEANUP, MasterRunner, LocalRunner

def checker(environment):
    while not environment.runner.state in [STATE_STOPPING, STATE_STOPPED, STATE_CLEANUP]:
        time.sleep(1)
        if environment.runner.stats.total.fail_ratio > 0.2:
            print(f"fail ratio was {environment.runner.stats.total.fail_ratio}, quitting")
            environment.runner.quit()
            return


@events.init.add_listener
def on_locust_init(environment, **_kwargs):
    # dont run this on workers, we only care about the aggregated numbers
    if isinstance(environment.runner, MasterRunner) or isinstance(environment.runner, LocalRunner):
        gevent.spawn(checker, environment)
```

# 参数化locustfiles

有两种主要方法可以对locustfile进行参数化。

## 基本环境变量

与任何程序一样，你可以使用环境变量：

在Linux/Mac上：

```bash
MY_FUNKY_VAR=42 locust ...
```

在windows上：

```bash
SET MY_FUNKY_VAR=42
locust ...
```

然后在你的locust测试文件中访问他们：

```python
import os
print(os.environ['MY_FUNKY_VAR'])
```

## 自定义参数

你可以使用 init_command_line_parser 事件向Locust添加自己的命令行参数。自定义参数也会在Web UI中展示并可编辑。如果为参数指定了选项，它们将在Web UI中以下拉菜单的形式展示。

```python
from locust import HttpUser, events, task


@events.init_command_line_parser.add_listener
def _(parser):
    parser.add_argument("--my-argument", type=str, env_var="LOCUST_MY_ARGUMENT", default="", help="It's working")
    # Choices will validate command line input and show a dropdown in the web UI
    parser.add_argument("--env", choices=["dev", "staging", "prod"], default="dev", help="Environment")
    # Set `include_in_web_ui` to False if you want to hide from the web UI
    parser.add_argument("--my-ui-invisible-argument", include_in_web_ui=False, default="I am invisible")
    # Set `is_secret` to True if you want the text input to be password masked in the web UI
    parser.add_argument("--my-ui-password-argument", is_secret=True, default="I am a secret")


@events.test_start.add_listener
def _(environment, **kw):
    print(f"Custom argument supplied: {environment.parsed_options.my_argument}")


class WebsiteUser(HttpUser):
    @task
    def my_task(self):
        print(f"my_argument={self.environment.parsed_options.my_argument}")
        print(f"my_ui_invisible_argument={self.environment.parsed_options.my_ui_invisible_argument}")
```

当运行分布式Locust时，自定义参数会在测试开始时自动转发给工作节点（但在那之前不会，所以你不能在测试实际开始之前依赖转发的参数）。
