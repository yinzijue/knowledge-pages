# 软件测试理论

## 现代定义
- 为了发现缺陷、评估质量和降低风险而执行程序的过程（在软件中识别其正确性、完整性、安全性和质量的过程）

## 7个核心原则
- 软件测试可以显示缺陷的存在，而不能证明其不存在
- 穷尽测试是不可能的
- 尽早测试，持续进行：在需求与设计阶段发现缺陷，降低修复成本
- 缺陷集群性：少数模块包含大部分被发现的缺陷(八二定律)
- 杀虫剂悖论：重复相同的测试用例，最终将不再能发现新的缺陷，需要更新用例，加入新的方法和场景
- 测试依赖于上下文：测试策略需要根据项目背景、风险、时间等调整
- 不存在缺陷的谬论：比起发现并修复大量缺陷，更重要的是验证软件是否满足用户的需求和期望

## 生命周期
- 软测流程
    - 需求分析
    - 测试计划
        - 定义策略、范围、资源、进度、风险
    - 测试设计与开发
        - 设计用例、准备测试数据、编写测试脚本
    - 测试环境搭建
    - 测试执行
    - 测试总结
- BUG生命周期
    - 新建
    - 指派
    - 修复
    - 验证
    - 关闭

## 测试级别
- 冒烟测试
    - 验证核心流程是否通畅
- 单元测试
    - 最小代码单元：函数、类等
- 集成测试
    - 模块、组件之间的接口和交互
- 系统测试
    - 完整的、集成的系统上测试
- 回归测试
    - 基于影响面，结合自动化
- 验收测试
    - α测试
        - 开发环境内部，模拟用户
    - β测试
        - 真实用户环境，真实用户
    - UAT
        - 用户验收测试
