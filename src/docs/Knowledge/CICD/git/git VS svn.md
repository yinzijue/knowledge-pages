# 集中式与分布式

## 集中式版本控制

svn采用的是集中式版本控制，即所有的版本库都是放在中央服务器中的，每次本地的修改都需要上传到中央服务器。

![[集中式版本控制]]

- 优点：

- 操作简单；
- 管理员可以轻松控制各开发者的权限；
- 维护方面，只需维护中央服务器上的数据库即可；

- 缺点：

- 在协同工作时可能造成提交文件不完整，版本库损坏等问题。例如各个版本之间相互依赖。
- 必须联网才能工作，无论局域网还是互联网，必须上网才能将本地版本推送至服务器进行保存；
- 依赖于中央服务器的稳定性，容易出现单点问题。

## 分布式版本控制

git采用的是分布式版本控制，支持离线操作。

- 优点：

- 去中心化设计，增强了系统的容错性，即使某台设备宕机，也可以在其他地方找到整个版本的完整备份；

- 缺点：

- 每个人都拥有全部代码，容易出现安全问题。
