# 基本名词解释

## Docker

docker是一款开源的容器化平台，它利用容器技术将应用程序及其依赖打包到一个可移植的容器中，使其能够在任何环境中运行而无需担心环境差异。

docker的核心组件包括**docker仓库、docker镜像、docker容器、docker引擎(docker客户端、docker守护进程、Docker REST API)**。

## 镜像

- 镜像是一个包含了容器运行时所需的依赖环境的静态文件。它是容器的模板。
- 容器镜像采用分层结构，用户制作镜像的每一步操作都会产生一个层——即一个增量rootfs。

## 容器

容器是独立运行的一个或一组应用，是镜像运行时的主体。容器之于镜像，类似面向对象中的实例之于类。

## 仓库

用于保存镜像的地方，类似于代码仓库。有公有仓库和私有仓库。

## DockerFile

DockerFile是一个包含了一系列构建镜像指令的文本文件。通过DockerFile可以定义一个完整的、可重复的Docker镜像构建过程。

## 引擎（低优先级）

Docker引擎是运行和管理容器的核心软件。主要组件包括Docker客户端、Docker守护进程、Docker REST API

# Docker优缺点

## 优点

1. 轻量级
2. 可移植性
3. 可靠性：提供了各理性和安全性，避免应用程序之间冲突和干扰
4. 简单易用
5. 可扩展性(集群管理，快速扩容)

## 缺点

1. 安全性：隔离不完全，可能受到攻击；
2. 存储：可能丢失或损坏，需备份；
3. 网络：网络通信可能受到限制或干扰；
4. 性能：需要性能优化和调整

# docker实现轻量化的原理

1. 镜像分层：(基础层、叠加层、可读层、可写层)未发生变化的层可以复用；
2. 基础镜像轻量化；
3. 构建过程优化：合并指令，优化构建步骤；
4. 清理不必要的文件和依赖；
5. 压缩镜像；

# 容器生命周期

- 创建(create，创建但不运行)
- 启动(start)
- 运行(run)
- 暂停/继续(可选)
- 停止(stop)
- 重启(restart)
- 销毁(rm)

# 容器化与虚拟化(基于Hypervisor)

- 架构

	- 容器化：OS级别的虚拟化，共享操作系统内核，每个容器只包含应用程序及其依赖；
	- 虚拟化：硬件虚拟化，模拟整个操作系统(硬件、操作系统、应用程序)

- 资源占用

	- 容器化：
	- 虚拟化：内存、CPU、存储；

- 启动时间

	- 容器化：启动应用程序及其依赖；
	- 虚拟化：启动整个操作系统；

- 隔离性

	- 容器化：不完全隔离；
	- 虚拟化：完全隔离，每个虚拟机有独立的操作系统和应用程序；

- 管理

	- 容器化：管理应用及其依赖
	- 虚拟化：管理多个操作系统和应用程序

## 虚拟技术选型

- 资源利用率、快速启动、高可移植：容器化
- 安全性、隔离性：虚拟机
