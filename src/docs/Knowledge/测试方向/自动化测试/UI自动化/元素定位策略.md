# 概述

selenium是一款web应用的自动化框架，可以让自动化程序像人一样在浏览器中操作web界面。selenium自动化流程大致如下：

1. 用户编写的自动化程序调用selenium客户端库函数(例如各种操作界面元素的方法)
2. 客户端发送selenium命令给浏览器驱动程序
3. 浏览器驱动程序接收命令，驱动浏览器执行命令
4. 浏览器执行命令
5. 浏览器驱动程序获取命令执行结果，返回给自动化程序
6. 自动化程序对返回结果进行处理

不同的浏览器需要用不同的驱动，以chrome为例，需要下载当前浏览器版本对应的chromedriver.exe文件，然后将文件路径写入自动化程序。

# 第一个程序

```python
from selenium import webdriver
from selenium.chrome.service import Service
driver_path=r'chromedriver.exe文件完整路径'
browser = webdriver.Chrome(service=Service(driver_path))
browser.get('https://www.baidu.com')
# 关闭浏览器窗口
driver.quit()
```

很久没关注selenium了，再次使用时版本来到了4.14.0。彼时写的UI自动化脚本一个比较头疼的问题在于驱动程序会随着浏览器版本更新以及脚本路径迁移而失效，需要人为地“自适应”。而不知从什么版本起，selenium已不再需要人为指定浏览器驱动程序路径，webdriver指定对应的浏览器即可自动识别驱动。上述代码中的驱动相关语句也就不再需要了。

如果你的设备因为种种原因无法使用较新版本selenium，并因此遇到了驱动更新方面的烦恼，建议使用python提供的webdriver-manager库，从而自动下载并使用浏览器驱动程序。

# webdriver常用属性及方法

|   |   |
|---|---|
|方法|作用|
|maximize/minimize/fullscreen_window()|最大化/最小化/全屏浏览器窗口|
|set_window_size(self, width, height, windowHandle: str = "current")|自定义窗口尺寸|
|get(url)|跳转到指定的URL，URL为字符串格式的地址|
|refresh()|刷新页面|
|back()|后退|
|forward()|前进|
|get_screenshot_as_png()|截图的base64内容|
|get_screenshot_as_file()|截图保存到文件|
|current_url|返回当前页面的url|
|page_source|当前页面源码|
|current_window_handle|当前窗口的句柄|
|window_handles|所有窗口的句柄|
|switch_to.alert|处理弹窗|
|switch_to.frame('frame_name')|切换框架|
|switch_to.window('main')|切换到main窗口|
