# 性能测试

## 压测流程
- 性能场景分析与创建
    - 压测场景
        - 业务峰值稳定性
            - 峰值业务稳定性考验
        - 新系统上线
            - 用户流量涌入涌入
        - 技术升级验证
            - 技术架构升级后进行性能评估
        - 容量规划
            - 为系统扩容、性能优化提供数据参考
        - 性能探测
            - 探测系统中的性能瓶颈点，进行针对优化
    - 压测策略
        - 负载测试
            - 不同负载下的性能表现，发现性能拐点，找出最佳性能(并发递增)
        - 压力测试
            - 高负荷运行及减小负载后系统的自我恢复能力
        - 基准测试
        - 配置测试
            - 找到性能最优的软硬件配置组合
        - 稳定性测试
            - 特定负载下持续运行，验证长期稳定性
    - 系统架构梳理(分析潜在风险点)
        - 端到端的请求链路
        - 技术架构
        - 分层结构
        - 模块划分
        - 数据库、缓存、消息队列等中间件使用
    - 性能场景设计
        - 业务范围
	        - 关键业务
	        - 访问量大
	        - 逻辑复杂
	        - 运营推广活动
        - 业务操作流程
        - 业务配比
	        - 分配线程到不同的业务
        - 准备工作
	        - 数据准备
	        - 机器准备
	        - 人员准备
	        - 全链路压测
    - 压测方案编写
- 压测脚本编写与调试
    - 编写
	    - 务必写上简明的断言，防止遗漏状态码为200的异常情况
    - 调试
	    - 查看结果树
    - 脚本执行
    - 指标监控
    - 定位瓶颈
	    - 硬件指标：CPU、memory、Disk I/O,Network I/O
	    - 中间件指标：线程池、连接池、JVM
	    - 数据库指标：慢查询、命中率、锁、参数设置
	    - 算法、缓冲、缓存、同步/异步
    - 性能调优
	    - 性能调优后必须进行功能回归测试
	    - 时空转换
    - 输出测试报告

## 面试实战
- 性能场景收集与分析&压测脚本编写及调试
- 脚本执行&指标监控
- 定位瓶颈&性能调优
