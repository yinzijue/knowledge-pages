# 接口测试

## 概念
- 不同系统或模块之间资源交流的方式

## 目的
- 越底层发现bug，修复成本越低；
- 不受前端干扰
- 检查系统的安全性和稳定性，前端的传参不可信；
- 降低测试成本（相对稳定，易于自动化）

## 分类
- 内部接口
- 外部接口
    - 调用外部接口
    - 提供给外部的接口

## 流程
- 查看API文档，了解接口信息(抓包)
    - 请求方法+接口地址+端口+路径+鉴权+入参+返参+错误码
- 编写测试用例及评审
    - 最少正向用例+最多方向用例
- 数据mock
- 使用测试工具
    - jmeter+ant+git+jenkins
- 出具接口测试报告(CI/CD)

## 要点
- 权限控制
    - 鉴权
        - 接口获取appid+secret
        - user+password
- 功能性
    - 参数正确性、完整性
    - 接口功能正确性、完整性
    - 输出结果的正确性
    - 异常情况容错处理
- 可靠性
- 性能
- 安全性
